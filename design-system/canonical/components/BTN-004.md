# Button Loading State Must Include Visually Hidden Loading Label

**Rule ID:** BTN-004
**Status:** draft
**Version:** 1.0.0
**Source:** Accepted feedback `fb-2026-05-01-btn-spinner-aria-label` — guardian accepted 2026-05-01; WCAG 2.1 SC 4.1.3 Status Messages; ARIA Authoring Practices Guide — Button pattern
**Scope:** `ui-lab/primitives/button/**`
**Priority:** high

## Do

When the Button is in loading state (`loading=true`), include a visually hidden text node inside the button that communicates the loading status to assistive technologies. The node must be visually hidden (not `display:none` or `visibility:hidden`, which would also hide it from screen readers) and must be adjacent to the spinner element.

```tsx
{loading && (
  <>
    <span className="btn__spinner" aria-hidden="true" />
    <span className="sr-only">Loading…</span>
  </>
)}
```

The `.sr-only` utility class must use the clip-path pattern to remain accessible to screen readers:

```css
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

The button must also retain `aria-busy="true"` while loading.

## Do Not

- Do not rely on `aria-busy="true"` alone without a status message node. Support for `aria-busy` varies across screen reader + browser combinations.
- Do not use `aria-label` on the spinner span. It is `aria-hidden`; an `aria-label` on a hidden element has no effect.
- Do not use `display: none` or `visibility: hidden` for the loading label — these remove the element from the accessibility tree.

## Why

WCAG 2.1 SC 4.1.3 requires that status messages (including dynamic loading states) be programmatically determinable without focus. A visually hidden "Loading…" node inside the button satisfies this requirement reliably. `aria-busy="true"` alone is insufficient because its screen reader support is inconsistent, particularly on older NVDA + Firefox combinations.

## Validation

Verify that when `loading=true`:
1. A text node with "Loading…" (or equivalent) is present inside the `<button>` element.
2. The text node has a class or style that visually hides it using the clip-path pattern.
3. The text node is NOT `aria-hidden="true"`.
4. `aria-busy="true"` is still present on the `<button>`.

## Impact

Affects `ui-lab/primitives/button/`. Loading stories in Storybook must include a screen reader audit step. The change is non-breaking: existing prop API is unchanged.

## Good Example

```tsx
<button
  type="button"
  className="btn btn--primary"
  aria-busy="true"
  aria-disabled={undefined}
>
  <span className="btn__spinner" aria-hidden="true" />
  <span className="sr-only">Loading…</span>
  <span className="btn__label">Submit</span>
</button>
```

## Bad Example

```tsx
<button
  type="button"
  className="btn btn--primary"
  aria-busy="true"
>
  <span className="btn__spinner" aria-hidden="true" />
  <span className="btn__label">Submit</span>
  {/* No visually hidden loading label — screen readers receive no status message */}
</button>
```
