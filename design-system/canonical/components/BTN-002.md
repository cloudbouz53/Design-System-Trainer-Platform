# Button Disabled State Uses Opacity

**Rule ID:** BTN-002
**Status:** active
**Version:** 1.0.0
**Source:** inferred-rules.md §Button disabled state; component-notes.md §Button §Interaction States; raw-observations.md §Button (disabled: opacity 0.4 observed) — human approval by guardian 2026-05-01
**Scope:** `ui-lab/primitives/button/**`
**Priority:** high

## Do

Use `opacity: 0.4` and `cursor: not-allowed` for the disabled state. Set `aria-disabled="true"` on the element.

```css
.btn[aria-disabled="true"],
.btn:disabled {
  opacity: 0.4;
  cursor: not-allowed;
  pointer-events: none;
}
```

## Do Not

Use a separate disabled background color token; opacity communicates the disabled state uniformly across all button variants without requiring duplicate token definitions.

```css
/* Bad */
.btn:disabled {
  background-color: var(--color-background-secondary); /* wrong approach */
  color: var(--color-text-secondary);                   /* wrong approach */
}
```

## Why

Opacity-based disabled state works across all button variants (primary, secondary, ghost) without requiring a separate token per variant. It visually communicates unavailability uniformly.

## Validation

Check that the disabled state sets `opacity` and `cursor: not-allowed`. Check that `aria-disabled="true"` is applied to the element when it is disabled (not just the `disabled` HTML attribute).

## Impact

Affects `ui-lab/primitives/button/`. Any button without a consistent disabled state fails the State Gate.

## Good Example

```tsx
<button
  aria-disabled={isDisabled}
  disabled={isDisabled}
  className={isDisabled ? 'btn btn--disabled' : 'btn'}
>
  {label}
</button>
```

```css
.btn--disabled,
.btn:disabled {
  opacity: 0.4;
  cursor: not-allowed;
}
```

## Bad Example

```tsx
<button style={{ backgroundColor: '#ccc', color: '#888' }} disabled>
  {label}
</button>
```
