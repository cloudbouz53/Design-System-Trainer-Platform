# Button Uses Semantic HTML Button Element

**Rule ID:** BTN-003
**Status:** active
**Version:** 1.0.0
**Source:** inferred-rules.md §Button uses semantic HTML button element; component-notes.md §Button §Accessibility; raw-observations.md §Button (uses `<button>` observed) — human approval by guardian 2026-05-01
**Scope:** `ui-lab/primitives/button/**`
**Priority:** critical

## Do

Use `<button>` element (with `type="button"` unless inside a form that needs `type="submit"`) for all interactive buttons.

```tsx
<button type="button" onClick={handleClick}>
  {label}
</button>
```

## Do Not

Use `<div>`, `<span>`, `<a>`, or other non-semantic elements as interactive buttons. If a link behavior is needed, use `<a href>` — not a styled `<button>`.

```tsx
/* Bad */
<div onClick={handleClick} role="button">{label}</div>
```

## Why

Native `<button>` provides keyboard interaction (`Enter`, `Space`), implicit `role="button"`, focus management, and form association for free. Re-implementing these on a `<div>` is error-prone and maintenance-heavy.

## Validation

Check the component implementation uses a `<button>` element as the root interactive element. If a `<div role="button">` is found, flag as a violation unless a documented exception is approved by the guardian.

## Impact

Affects `ui-lab/primitives/button/` and any component that wraps or extends Button. Non-semantic implementation fails both the Accessibility Gate and this rule.

## Good Example

```tsx
function Button({ label, onClick, disabled }: ButtonProps) {
  return (
    <button type="button" onClick={onClick} disabled={disabled}>
      {label}
    </button>
  );
}
```

## Bad Example

```tsx
function Button({ label, onClick }: ButtonProps) {
  return (
    <div onClick={onClick} className="btn">
      {label}
    </div>
  );
}
```
