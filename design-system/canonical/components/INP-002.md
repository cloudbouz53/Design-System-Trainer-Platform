# Input Error Message Uses aria-describedby

**Rule ID:** INP-002
**Status:** active
**Version:** 1.0.0
**Source:** inferred-rules.md §Input error message uses aria-describedby; component-notes.md §Input §Accessibility; raw-observations.md §Input — human approval by guardian 2026-05-01
**Scope:** `ui-lab/primitives/input/**`
**Priority:** critical

## Do

When an input is in the error state:
1. Set `aria-invalid="true"` on the `<input>`.
2. Render the error message in a separate element with a unique `id`.
3. Add `aria-describedby="<error-element-id>"` to the `<input>`.
4. Mark the error element with `role="alert"` or ensure it is in an `aria-live` region.

```tsx
<input
  id="email-input"
  aria-invalid={hasError}
  aria-describedby={hasError ? 'email-error' : undefined}
/>
{hasError && (
  <span id="email-error" role="alert">
    {errorMessage}
  </span>
)}
```

## Do Not

Only apply a red border style without programmatically linking the error message to the input.

```tsx
/* Bad — visual only */
<input className={hasError ? 'input-error' : ''} />
{hasError && <span>{errorMessage}</span>}
// Missing aria-invalid, aria-describedby, and role="alert"
```

## Why

Screen reader users cannot see a red border. Without `aria-invalid` and `aria-describedby`, they have no way to discover the error description or know the field is in error.

## Validation

Check that:
1. Error state sets `aria-invalid="true"` on the `<input>`
2. `aria-describedby` points to an element with a matching `id`
3. The referenced element has `role="alert"` OR is inside an `aria-live` region

## Impact

Affects `ui-lab/primitives/input/`. Failure immediately fails the Accessibility Gate.

## Good Example

```tsx
<input
  id="email"
  type="email"
  aria-invalid="true"
  aria-describedby="email-error"
/>
<span id="email-error" role="alert">
  Please enter a valid email address.
</span>
```

## Bad Example

```tsx
<input type="email" className="border-red" />
<span className="text-red">Invalid email</span>
{/* No aria-invalid, no aria-describedby, no role — fails INP-002 */}
```
