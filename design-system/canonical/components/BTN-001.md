# Button Focus Ring

**Rule ID:** BTN-001
**Status:** active
**Version:** 1.0.0
**Source:** inferred-rules.md §Button focus ring; component-notes.md §Button §Accessibility; raw-observations.md §Button (focus observed) — human approval by guardian 2026-05-01
**Scope:** `ui-lab/primitives/button/**`
**Priority:** critical

## Do

Apply a visible focus ring on `:focus-visible` using the interactive color token:

```css
button:focus-visible {
  outline: 2px solid var(--color-interactive);
  outline-offset: 2px;
}
```

## Do Not

Remove or suppress the focus ring with `outline: none` or `outline: 0` on focused or focus-visible buttons.

```css
/* Bad */
button:focus { outline: none; } /* removes accessibility affordance */
```

## Why

Visible focus indicators are required for keyboard navigation and assistive technology users. WCAG 2.4.7 requires keyboard focus to be visible. Without a focus ring, keyboard users cannot determine which button has focus.

## Validation

Check that `:focus-visible` styles include `outline` with a non-transparent, non-zero value using `--color-interactive` or equivalent. Check that no `:focus` rule sets `outline: none` or `outline: 0` without a `:focus-visible` replacement.

## Impact

Affects `ui-lab/primitives/button/` and any component set or app that uses the Button primitive. Removing the focus ring fails the Accessibility Gate.

## Good Example

```css
.btn:focus-visible {
  outline: 2px solid var(--color-interactive);
  outline-offset: 2px;
}
```

## Bad Example

```css
.btn:focus {
  outline: none; /* keyboard users cannot see focus — fails BTN-001 */
}
```
