# Use Spacing Tokens on 4px Grid

**Rule ID:** TOK-002
**Status:** active
**Version:** 1.0.0
**Source:** inferred-rules.md §Use spacing tokens; materials-notes.md §Spacing; raw-observations.md §Spacing — human approval by guardian 2026-05-01
**Scope:** `ui-lab/**`
**Priority:** high

## Do

Use spacing tokens (`--spacing-1` through `--spacing-16`) for all margin, padding, and gap values.

```css
/* Good */
padding: var(--spacing-2) var(--spacing-3);
gap: var(--spacing-4);
margin-bottom: var(--spacing-6);
```

## Do Not

Use arbitrary pixel values that are not multiples of 4px (e.g., `11px`, `13px`, `15px`, `20px`).

```css
/* Bad */
padding: 11px 14px;  /* not on 4px grid — fails TOK-002 */
gap: 20px;           /* not a spacing token — fails TOK-002 */
```

## Why

A consistent 4px spatial grid produces visual coherence and predictable layout. Arbitrary values create visual noise and make spacing choices non-reproducible.

## Validation

Grep for `padding:`, `margin:`, `gap:`, `top:`, `bottom:`, `left:`, `right:` values that contain pixel values not matching `var(--spacing-*)` or `0`. Allowed: `0`, `100%`, `auto`, `var(--spacing-*)`.

## Impact

Affects all layout and spacing in `ui-lab/`. Drift in spacing reduces visual consistency across components and apps.

## Good Example

```css
.button {
  padding: var(--spacing-2) var(--spacing-3);
}
.form-group {
  gap: var(--spacing-4);
}
```

## Bad Example

```css
.button {
  padding: 8px 13px; /* 13px is not on the 4px grid — fails TOK-002 */
}
```
