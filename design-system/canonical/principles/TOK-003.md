# Use Border Radius Tokens

**Rule ID:** TOK-003
**Status:** active
**Version:** 1.0.0
**Source:** inferred-rules.md §Use border-radius tokens; materials-notes.md §Border Radius; raw-observations.md §Border Radius — human approval by guardian 2026-05-01
**Scope:** `ui-lab/**`
**Priority:** medium

## Do

Use `--radius-sm` (4px), `--radius-md` (6px), `--radius-lg` (8px), or `--radius-full` (9999px) for all `border-radius` values.

```css
/* Good */
border-radius: var(--radius-md);
```

## Do Not

Use arbitrary border-radius values like `5px`, `7px`, `10px`, or `100%` (use `--radius-full` instead of `100%`).

```css
/* Bad */
border-radius: 5px;   /* not a radius token — fails TOK-003 */
border-radius: 10px;  /* not a radius token — fails TOK-003 */
```

## Why

Consistent radii produce a unified visual language. Three distinct values (sm/md/lg) are enough to express the full range of component shapes without creating visual inconsistency.

## Validation

Grep for `border-radius:` values containing px units or percentage values not matching `var(--radius-*)`. Allowed: `0`, `var(--radius-*)`.

## Impact

Affects all components and apps in `ui-lab/`. Inconsistent radii create visual fragmentation.

## Good Example

```css
.input {
  border-radius: var(--radius-md); /* 6px */
}
.card {
  border-radius: var(--radius-lg); /* 8px */
}
.pill {
  border-radius: var(--radius-full); /* 9999px */
}
```

## Bad Example

```css
.input {
  border-radius: 5px; /* arbitrary — fails TOK-003 */
}
```
